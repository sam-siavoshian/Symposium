**How it works:**

- **Line 1 `//\`** — In C++, `\` at end of a `//` comment is line continuation, so line 2 is also commented out. In Rust, `\` is not line continuation, so only line 1 is a comment.
- **Line 2** — Rust sees the `fn main()` with Fibonacci logic, then `/*` opens a block comment. C++ sees nothing (still in the `//` comment from line 1).
- **Lines 3–5** — C++ sees its `#include` directives and `int main()`. Rust sees block comment text (no `/*` or `*/` in the C++ code to interfere with nesting).
- **Line 6 `// */`** — C++ sees a line comment. Rust sees `*/` which closes the block comment (the space between `//` and `*/` prevents Rust from parsing `/*` as a nested comment opener).
